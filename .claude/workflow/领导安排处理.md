# Workflow：领导安排处理

> 收到上级或管理岗工作安排时的标准处理流程

---

## 输入

- 安排来源（王敬宇/董文静/靳攀/王超等）
- 渠道（钉钉群/电话/当面/群通知）
- 任务描述

## 流程

### Step 1：识别来源与层级
```
判断：
  → 直接上级（王敬宇）→ 优先处理
  → 包保人（董文静）→ 按期执行
  → 管理岗（靳攀/王红义）→ 配合执行
  → 安全/资产（王亮/荆幸斌）→ 专项对接
```

### Step 2：任务四层拆解
```
① 表层任务：明确要求做什么
② 隐含目标：【推测】领导真正关心什么
③ 风险重点：什么情况会出问题
④ 验收标准：怎么做算完成
```

### Step 3：拆分子项
```
分解为可执行的动作：
  → 谁做（责任人）
  → 做什么（具体事项）
  → 什么时候做完（截止时间）
  → 需要什么资源/信息
```

### Step 4：更新记录
```
写 memory/tasks.md
  → 新增待办项
写 context/today.md
  → 更新今日重点/风险
```

### Step 5：跟踪闭环
```
按领导类型选择汇报策略：
  → 紧急强化型（王敬宇？）→ 即刻确认+首次快报
  → 稳定推进型 → 按节点汇报
  → 高关注检查型 → 分阶段主动同步
```

---

## 检查清单

- [ ] 任务来源是否明确（谁安排的）？
- [ ] 是否拆解到可执行的动作？
- [ ] 是否指定了责任人和截止时间？
- [ ] 是否更新了tasks.md？
- [ ] 是否需要更新context/today.md？
- [ ] 是否需要回复确认收到？
