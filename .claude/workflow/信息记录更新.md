# Workflow：信息记录与更新

> 每次对话结束后更新记忆/任务/上下文的标准流程

---

## 输入

- 本轮对话中产生的所有新信息

## 流程

### Step 1：分类判断

```
本次对话产生了什么？
  ├─ 新任务/新截止日 → 更新tasks.md
  ├─ 人员新观察 → 更新 references/人员档案.md
  ├─ 领导新安排 → 更新 tasks.md + leadership.md
  ├─ 新经验/新规则 → 更新 experience.md
  ├─ 犯了错 → 更新 mistakes.md
  ├─ 改变了今日状态 → 更新 context/today.md
  └─ 重要决策/事件 → 更新 history.md
```

### Step 2：先读后写
```
⚠️ 写入前必须Read目标文件获取最新内容
  → 不可依赖上下文中的缓存版本
  → 确认内容尚未存在再写入，避免重复
```

### Step 3：写入规范
```
  → 只写变化部分，不重写全文
  → 保留原有内容
  → 标注日期 YYYY-MM-DD
  → 信息密度优先（key:value 格式）
```

### Step 4：检查索引
```
如果新建了 memory/ 文件
  → 必须在 MEMORY.md 添加一行索引
```

---

## 检查清单

- [ ] 是否有新任务需要更新？
- [ ] 是否有新经验值得记录？
- [ ] 是否犯了需要记录的错误？
- [ ] 今日状态是否有变化？
- [ ] 写入前是否Read了目标文件最新内容？
- [ ] 是否避免了重复写入？

## 关联

- Memory: 全部
- Knowledge: 无
