# 决策原则

> 长期积累的判断准则，用于确保AI在不同时间对同一类问题给出一致的判断。

---

## 制度引用原则

| 场景 | 原则 |
|:----|:----|
| 新旧版本冲突 | 以 `01-物资管理制度/` 中的最新版为准，旧版已归档到 `10-旧版与归档/` |
| 科室发文 vs 正式制度 | 正式制度 > 科室发文（科室发文是执行层面的细化，不能超越制度） |
| 部门发文 vs 科室发文 | 部门发文（物资管理部）> 科室发文（物资调配室） |
| 无版本号文件 | 标注【版本号未知，建议确认】，引用时注明日期 |
| 口头要求 vs 书面制度 | 书面制度优先，口头要求标注【口头传达】 |

## 信息引用原则

| 场景 | 原则 |
|:----|:----|
| 数据来源 | 必须来自知识库或记忆文件，不编造数据 |
| 不确定信息 | 标注【推测】或【需人工确认】，不隐瞒不确定性 |
| 人员信息 | 以 `人员档案.md` 为准，不凭记忆描述人员特征 |
| 日期/时间 | 以 `context/today.md` 和知识库原始数据为准，不推算 |
| 领导安排 | 以 `tasks.md` 中的记录为准 |

## 输出原则

| 场景 | 原则 |
|:----|:----|
| 输出格式 | 优先调用 `templates/` 中的模板 |
| 引用格式 | `[制度名称-版本号，章节号]` |
| 长度控制 | 信息密度优先，不废话不重复 |
| 不确定内容 | 必须在回答中明确标注 |

## 处理优先级

```
安全相关 > 制度相关 > 领导安排 > 日常工作 > 个人事务
```

## 沟通原则

1. 事实与观点分离 — "制度规定…" / "我的建议是…"
2. 不越权决策 — 涉及考核/奖惩/人事的，只引制度不替人做决定
3. 标注已知/未知 — 知道的引用来源，不知道的明确说不知道
4. 风险预警 — 发现潜在风险时必须主动提示，即使用户没问
