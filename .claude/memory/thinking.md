# 思考流程

> 收到任何任务/问题后的标准思考路径。不是知识，是流程。

---

## 通用思考链（八步）

```
收到输入
  ↓
① 判断类型
   ├─ 询问类（查制度/查人员/查历史）
   ├─ 执行类（写安排/写日报/写纪要）
   ├─ 记录类（更新任务/记录事件）
   └─ 分析类（制度分析/风险判断）
  ↓
② 读取 Memory
   ├─ identity → 以谁的身份回答
   ├─ mistakes → 类似问题以前犯过错吗
   └─ experience → 有没有历史经验可用
  ↓
③ 读取 Context
   ├─ today → 今天日期/值班/重点/截止
   └─ 最近领导安排 → 有没有相关背景
  ↓
④ 关联 Skill（如果适用）
   ├─ 涉及哪个Skill
   └─ 调Skill的rules/checklist
  ↓
⑤ 检索 Knowledge
   ├─ 查 INDEX.md → 定位文件
   └─ 读对应文件 → 获取准确信息
  ↓
⑥ 引用 Template（如果输出）
   ├─ 匹配最近的模板
   └─ 填充内容
  ↓
⑦ 生成回答
   ├─ 引用来源标注
   ├─ 不确定内容标注【推测】/【需确认】
   └─ 检查是否有遗漏
  ↓
⑧ 判断更新
   ├─ 是否产生新任务 → 更新任务与记录
   ├─ 是否产生新经验 → 更新 experience.md
   ├─ 是否犯了错 → 更新 mistakes.md
   └─ 是否改变了今日状态 → 更新 context/today.md
```

---

## 按类型细化

### 询问类（查制度/人员/历史）
```
① 用户在问什么 → 精确理解需求
② 读 context/today → 是否有相关背景
③ 读 INDEX.md → 定位文件
④ 读目标文件 → 提取准确信息
⑤ 交叉验证（制度版本/关联文件）
⑥ 引用来源回答
⑦ 不确定 → 标注【需确认】
```

### 执行类（写安排/写日报）
```
① 确认时间范围（哪一天/哪一周）
② 读 context/today → 今日状态
③ 读 Memory → 人员特点/领导偏好
④ 读对应 Workflow → 按步骤执行
⑤ 读对应 Template → 按模板输出
⑥ 输出后更新 context/today
```

### 记录类（更新任务/记录事件）
```
① 读目标文件当前内容（Read）
② 对比新信息 → 哪些是新增的
③ 写入（Edit） → 保留原有内容，只改变化部分
④ 更新 context/today（如果涉及今日状态）
```

### 分析类（制度分析/风险判断）
```
① 明确分析对象
② 索引知识库定位相关制度/文件
③ 对比历史经验（experience.md）
④ 引用决策原则（decision.md）
⑤ 输出结构化分析（背景/依据/结论/建议）
```

## 必须遵守的规则

1. **先读后写** — Edit前必须Read目标文件最新状态
2. **查数据不推算** — 值班表/人员信息等必须以原始数据为准，不推理估算
3. **标注来源** — 引用制度必须标注版本号
4. **不确定要说明** — 没有依据的标注【推测】，无法判断的标注【需人工确认】
5. **先查INDEX.md** — 不要在知识库里盲目搜索，先读索引
6. **每轮结束检查更新** — 新任务/新经验/新错误 → 写入对应文件
