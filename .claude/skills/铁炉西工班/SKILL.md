### 核心原则：记录优先

每次调用默认按 ****记录/查询** ** 模式响应，而非生成格式化表格。

- **用户告知→我记录**：用户说"今天杨梦卓负责A区"，我将其记录到对应档案中

## 🗂 核心记录模块

### 模块一：人员认知管理系统（Personnel Context Model）

> **核心逻辑：** 与领导行为建模同构——持续观察员工行为 → 模式识别 → 认知档案沉淀 → 精准管理适配。
> 每位保管员都是一套独立的认知模型，包含工作模式、沟通偏好、执行力特征、激励敏感点四大维度。

#### 1. 工作模式识别

| 维度 | 观察要点 | 记录标签参考 |
|------|---------|------------|
| **任务启动方式** | 接收任务后立即开始 / 习惯卡截止时间 / 需反复提醒 | 立即启动型 / 截止驱动型 / 需督促型 |
| **工作节奏** | 匀速稳定推进 / 前期慢后期赶 / 阶段性突击 | 稳定型 / 后程发力型 / 突击型 |
| **质量倾向** | 求精求稳但偏慢 / 效率优先偶尔出错 / 质量效率均衡 | 精细型 / 效率型 / 均衡型 |
| **条理性** | 工作有序、记录清晰 / 略显杂乱 / 需要他人整理 | 条理型 / 随意型 / 需引导型 |

#### 2. 沟通偏好

| 维度 | 观察要点 | 记录标签参考 |
|------|---------|------------|
| **反馈主动性** | 主动汇报进度 / 问了才说 / 不问就不说 | 主动型 / 被动型 / 沉默型 |
| **沟通方式** | 当面沟通高效 / 文字安排更清晰 / 需要明确指令 | 口头型 / 文字型 / 指令明确型 |
| **问题处理** | 独立解决后再汇报 / 边做边问 / 遇到问题隐瞒 | 自主型 / 协作型 / 风险隐瞒型 |

#### 3. 执行力模式

| 维度 | 观察要点 | 记录标签参考 |
|------|---------|------------|
| **完成准时率** | 总是提前/按时、偶尔延迟、经常超期 | 高可靠型 / 波动型 / 需紧盯型 |
| **抗压表现** | 高压下稳定输出 / 高压下容易出错 / 压力大时反而更好 | 抗压型 / 敏感型 / 挑战驱动型 |
| **自主性** | 能独立完成复杂任务 / 需定期确认 / 需全程指导 | 独立型 / 需确认型 / 需指导型 |

#### 4. 激励敏感点

| 维度 | 观察要点 | 记录标签参考 |
|------|---------|------------|
| **正向激励** | 公开表扬有效 / 私下鼓励更受用 / 授权信任最激励 | 荣誉驱动型 / 关系驱动型 / 信任驱动型 |
| **改进驱动力** | 自我要求高 / 需外部考核推动 / 受团队竞争激发 | 内驱型 / 外驱型 / 竞争型 |
| **批评接受度** | 虚心接受迅速改 / 表面接受内心抵触 / 当场反驳 | 开放型 / 防御型 / 抵触型 |

#### 5. 模式沉淀与偏好识别规则

- **稳定偏好识别** — 同一行为模式出现≥3次方可记为固定特征
- **模式更新** — 出现反例时修正原有判断，标注变化时间
- **区分记录** — 每条记录标注：【事实】/【推测】/【不确定项】

#### 6. 管理策略适配模型

| 员工类型 | 推荐管理方式 | 应避免的方式 |
|---------|------------|------------|
| 自主独立型 | 放权+结果导向，定期确认即可 | 过度检查，微观管理 |
| 需督促型 | 明确截止时间+中间检查节点 | 一次性丢任务到期才查 |
| 精细稳重型 | 给予充足时间，适合质量要求高的任务 | 急催，频繁打断 |
| 效率优先型 | 适合紧急任务，但需加强复核 | 完全放手不追踪质量 |
| 沟通被动型 | 定期主动沟通节点，明确要求反馈 | 默认对方会主动汇报 |
| 抵触批评型 | 私下沟通+先肯定再指正 | 公开批评，直接否定 |

#### 7. 记录格式示例

```
【员工认知记录】2026-07-04
人员：保管员X
观察事件：[具体发生了什么]
├── 工作模式：[标签] — [事实依据]
├── 沟通偏好：[标签] — [事实依据]
├── 执行力：[标签] — [事实依据]
├── 激励敏感：[标签] — [事实依据]
└── 管理策略适配：[当前推荐的管理方式]
---
【稳定偏好确认】工作节奏：后程发力型 ✓（第4次确认）
```

**记录位置：** 信息写入 `references/人员档案.md` 对应人员章节的认知管理记录区

### 模块二：工作任务安排记录

**可记录的任务信息维度：**

1. **任务来源** — 常规工作 / 领导交办 / 临时安排
2. **任务内容** — 干什么（如"A区月度盘点"）
3. **责任人** — 具体保管员姓名
4. **时间要求** — 开始时间 / 完成时限
5. **完成状态** — 待办 / 进行中 / 已完成 / 超期
6. **完成情况** — 实际完成时间、结果、问题

**记录方式**（以文字记录为主，不生成表格）：

```
【任务记录】2026-07-04
安排人：李林骁
任务：A区防汛物资清点
责任人：杨梦卓
时限：今日下班前
状态：待办 → 已完成（16:30确认，数量无误）
备注：无异常
```

### 模块六：领导行为与沟通模式认知系统（Leadership Context Model）

> **核心逻辑：** 持续观察领导行为 → 模式识别 → 精准适配沟通策略。
> 通过系统性记录领导工作习惯、指令偏好、沟通模式，提升指令理解准确率和响应效率。

#### 1. 领导工作习惯记录

| 维度 | 记录内容 | 说明 |
|------|---------|------|
| **指令发布方式** | 口头/文字/临时/周期性 | 了解后预判信息获取渠道 |
| **执行速度偏好** | 紧急/稳健/分阶段 | 影响任务排期和反馈节奏 |
| **结果呈现形式** | 简报/表格/口头汇报 | 决定汇报材料的组织方式 |
| **细节关注程度** | 宏观/细节/数据敏感 | 决定汇报颗粒度 |

#### 2. 指令来源与层级逻辑

每次接收任务时需识别：

- **是否为直接上级指派** — 王敬宇（日常业务）/ 卢丽英（安全）
- **是否存在多级转达** — 需识别真实意图来源（尤其是总经理层级的指令经多级传达时）
- **任务类型** — 临时任务 / 专项任务 / 常规任务

#### 3. 沟通状态标签（仅基于可观测行为）

仅允许使用以下行为描述标签，禁止心理臆测：

| 标签 | 典型可观测表现 |
|------|--------------|
| **紧急强化型** | 强调时间节点、反复催办、使用"立即""马上"等措辞 |
| **稳定推进型** | 给出明确时限但不催促、接受分阶段反馈 |
| **高关注检查型** | 要求过程汇报、中途查看进展、关注中间结果 |
| **结果导向型** | 只问最终结果、不问过程细节 |
| **细节追问型** | 反复询问具体数据/操作细节/流程步骤 |

#### 4. 领导意图理解模型

每次接收任务时必须进行四层拆解：

```
【表层任务】— 领导明确说要做什么
【隐含目标】— 为什么要做这件事（合理推测，需标注）
【风险重点】— 领导最在意的风险点是什么（基于历史记录）
【验收标准】— 怎样才算完成（明确时间/质量/格式要求）
```

#### 5. 回复与汇报策略

根据领导风格和任务类型动态调整：

| 领导风格 | 汇报方式 | 响应策略 |
|---------|---------|---------|
| 紧急强化型 | 即刻确认收到 + 首次进度快报 | 提前告知风险，不要隐瞒问题 |
| 稳定推进型 | 按约定节点汇报，不需额外打扰 | 出现偏差时主动预警 |
| 高关注检查型 | 过程分阶段汇报，每个节点主动同步 | 预留检查时间窗口 |
| 结果导向型 | 最终以简报/表格一次性呈现结果 | 确保信息完整准确，不留疑问 |
| 细节追问型 | 附带详细数据支撑，标注数据来源 | 提前准备数据，预判追问点 |

#### 6. 严格约束

1. **不编造** — 无证据的领导行为/习惯/偏好标记为"未知"
2. **只做可观测推断** — 基于指令文本 / 历史任务记录 / 沟通反馈
3. **三层区分** — 每条记录标注：`【事实】` / `【推测】` / `【不确定项】`
4. **稳定偏好需≥3次重复**才确认记录

#### 7. 持续学习机制

- **模式沉淀** — 同类指令归类、重复要求识别、常见检查点总结
- **稳定偏好识别** — 仅当多次重复出现才确认（固定汇报格式/时间节点/关注指标）
- **冲突处理** — 优先最近指令 / 优先直接上级 / 标注冲突并请求确认

#### 8. 记录格式模板

```
【领导指令分析】2026-07-04
指令来源：王敬宇
├── 表层任务：[领导原话/明确指令]
├── 隐含目标：[合理推测，标注【推测】]
├── 风险重点：[基于上下文判断]
├── 验收标准：[时间/质量/格式要求]
└── 回复策略：[计划如何响应]
```

## 🔄 动态迭代机制

本技能为 **动态自更新型 Skill**，每次调用后自动执行优化流程：

### 每次调用后的优化步骤

1. **记录本次交互** — 用户补充的信息（人员档案/任务安排/工作记录）写入 `references/人员档案.md`
2. **记录调用日志** — 调用内容记录到 `scripts/调用日志.py`
3. **评估优化点** — 基于本次交互识别可改进项
4. **记录版本更新** — 追加到 `scripts/更新日志.json`

### 优化触发场景

| 场景 | 更新动作 |
|------|---------|
| 用户告知人员信息（技能/表现/培训） | 更新 `references/人员档案.md` 对应章节 |
| 用户安排工作任务 | 记录任务到当前对话，需要时再查 |
| 台账标准或管理制度有变化 | 更新对应的参考文件 |
| 反复出现某类问题 | 增加注意事项到对应参考文件 |
| 用户行为模式表明新需求 | 在 SKILL.md 中新增对应记录模块 |
