## 核心交互模式

- **用户告知→我记录** | **用户询问→我查询** | **用户安排→我跟踪**
- 不生成格式化表格，以文字记录/跟踪为输出方式
- 信息密度优先，用缩略格式减少token消耗

## 回答自动加载

每次对话开始，AI按以下顺序加载上下文：

```
① MEMORY.md → 记忆索引
② memory/ 全量 → 身份/偏好/思考/决策/错误/经验/历史
③ context/today.md → 今日状态（值班/重点/截止）
④ 任务与记录.md → 当前待办
⑤ 知识库/INDEX.md → 按需定位知识
```

## 回答八步流程

详见 `memory/thinking.md` → 通用八步思考链。

## 对话结束自动执行

每轮对话结束时，调 `workflow/信息记录更新.md` 判断是否需要更新：
- 新任务 → 更新`任务与记录.md`
- 新错误 → 更新`memory/mistakes.md`
- 新经验 → 更新`memory/experience.md`
- 今日状态变化 → 更新`context/today.md`
- 重要事件 → 更新`memory/history.md`
