## 一、人员认知管理（4维度 + 策略映射）

持续观察→模式识别→档案沉淀→管理适配。

### 工作模式
任务启动（立即/截止/需督促）| 节奏（稳定/后程发力/突击）| 质量（精细/效率/均衡）| 条理（条理/随意/需引导）

### 沟通偏好
反馈（主动/被动/沉默）| 方式（口头/文字/指令明确）| 问题处理（自主/协作/隐瞒）

### 执行力
准时率（高可靠/波动/需紧盯）| 抗压（抗压/敏感/挑战驱动）| 自主性（独立/需确认/需指导）

### 激励敏感
正向（荣誉驱动/关系驱动/信任驱动）| 改进动力（内驱/外驱/竞争型）| 批评接受（开放/防御/抵触）

### 管理策略映射
| 类型 | 推荐 | 避免 |
|:----|:----|:----|
| 自主独立型 | 放权+结果导向 | 微观管理 |
| 需督促型 | 明确截止+中间检查 | 到期才查 |
| 精细稳重型 | 给充足时间 | 急催打断 |
| 效率优先型 | 适合紧急任务+复核 | 完全放手 |
| 沟通被动型 | 定期主动沟通 | 默认会主动汇报 |
| 抵触批评型 | 私下先肯定再指正 | 公开批评 |

### 稳定偏好规则
- ≥3次重复方可确认固定特征 | 反例出现时修正并标注时间
- 每条标注：【事实】/【推测】/【不确定项】
- 记录格式：`【员工认知记录】YYYY-MM-DD \n 人员： \n 观察事件： \n ├工作模式： \n ├沟通： \n ├执行力： \n ├激励： \n └策略：`
- 写入 `references/人员档案.md` 对应章节

## 六、领导行为认知系统

### 沟通标签（仅可观测行为）
紧急强化型（反复催办/限时）| 稳定推进型（给时限不催）| 高关注检查型（过程汇报）| 结果导向型（只看结果）| 细节追问型（数据细节）

### 意图四层拆解
```
【表层任务】 | 【隐含目标】(标注【推测】) | 【风险重点】 | 【验收标准】
```

### 汇报策略
| 领导型 | 汇报方式 |
|:------|:--------|
| 紧急强化 | 即刻确认+首次快报，提前说风险 |
| 稳定推进 | 按节点汇报，偏差时主动预警 |
| 高关注检查 | 分阶段主动同步，留检查窗口 |
| 结果导向 | 一次性完整简报 |
| 细节追问 | 附数据来源，预判追问点 |

### 约束
- 不编造，无证据标记"未知"
- 仅做可观测推断（指令文本/历史任务/沟通反馈）
- 每条标注【事实】/【推测】/【不确定项】
- 稳定偏好≥3次重复确认
- 冲突时：优先最近指令 > 优先直接上级 > 标注并请求确认

### 记录格式
```
【领导指令分析】YYYY-MM-DD 来源：[姓名]
├ 表层： | ├ 隐含：【推测】 | ├ 风险： | ├ 验收： | └ 策略：
```

## 记忆机制（每轮对话自动触发）

每轮对话结束后，按以下层级自动执行记忆存储。核心原则：重要信息不过夜，每轮都落地。

### 层级一：每轮对话必做（轻量级）

| 触发条件 | 动作 | 写入位置 |
|---------|------|---------|
| 每轮对话结束 | 记录调用日志 | `scripts/调用日志.py add` |
| 产生新任务、截止日变更 | 更新待办清单 | `.claude/memory/任务与记录.md` |
| 产生新的工作任务记录 | 写入任务记录 | 按SKILL §二格式写入`references/人员档案.md`对应章节 |

### 层级二：新工作安排时做（中量级）

| 触发条件 | 动作 | 写入位置 |
|---------|------|---------|
| 领导交办新任务 | 指令拆解+跟踪 | `references/领导认知档案.md` + `.claude/memory/任务与记录.md` |
| 新工作分配给人 | 同步更新待办 | `.claude/memory/任务与记录.md` |
| 库区发生变动（物资挪动/新物资进场） | 记录变动 | `.claude/memory/任务与记录.md` 或新建专项记忆 |
| 值班表/库区分配更新 | 更新记忆 | `.claude/memory/值班表与库区轮换管理.md` |

### 层级三：重要信息出现时做（重量级）

| 触发条件 | 动作 | 写入位置 |
|---------|------|---------|
| 观察到人员行为模式（≥1次显著行为） | 写入人员认知档案 | `references/人员档案.md` 对应章节（标注【事实】/【推测】） |
| 观察到领导行为模式（≥1次显著行为） | 写入领导认知档案 | `references/领导认知档案.md` 对应章节 |
| 出现新的专项工作 | 建立专项记忆文件 | `.claude/memory/` 新建文件 + MEMORY.md 加索引 |
| 用户提供重要背景/规则 | 保存为记忆 | `.claude/memory/` 对应文件 |
| 用户明确说"记住" | 立即写入+加索引 | `.claude/memory/` 对应文件 + MEMORY.md 加索引 |

### 记忆写入规范

1. **不要重复已有内容** — 写入前检查目标文件中是否已有相同信息，有则更新而非追加
2. **标注置信度** — 人员/领导行为记录使用【事实】/【推测】/【不确定】标注
3. **附带时间** — 每条记录标注日期 YYYY-MM-DD
4. **更新索引** — 新建 `.claude/memory/` 文件后，必须在 MEMORY.md 添加一行索引
5. **推送到远端** — 更新记忆后如涉及跨设备同步，提醒用户 `git add+commit+push`
6. **先检查后写入** — 避免重复文件，同一主题覆盖更新而非新建

### 触发时序

```
用户输入 → 我回复（记录/查询/跟踪） 
         → 每轮对话结束：
             必做：调用日志 + 检查是否有新任务/截止日变化
             如有新工作：更新待办清单 + 对应档案
             如有重要信息：写入记忆文件 + 更新索引
         → 用户确认后推送（如有需要）
```
