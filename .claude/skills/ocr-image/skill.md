# 图片识别录入

当用户发送图片或要求识别图片时，执行以下流程：

## 步骤1：识别图片

运行OCR辅助脚本：

```bash
/c/Users/cha18/AppData/Local/Programs/Python/Python312/python.exe -X utf8 "F:/学习/串串香支出与利润/ocr_helper.py" <图片路径1> [图片路径2] ...
```

脚本会自动：
- 使用RapidOCR识别中文内容
- 提取商品名和金额（匹配 ￥/¥ 价格模式）
- 同批次图片内自动去重（商品名+价格完全一致视为重复）
- 金额四舍五入到元
- 推断子类（调味料→食材、签签→耗材、冷藏柜→装修等）

## 步骤2：展示结果并确认

识别完成后，向用户展示：
1. 每张图片的原始识别文本（供用户核实）
2. 去重后的条目清单（序号、商品名、子类、金额）
3. 合计金额

**重要：必须等用户确认后才录入，不要自动录入。**

如果用户还提供了文字描述的收支（如"冷藏柜3080元"），将文字信息与图片识别结果合并后一起展示，统一确认。

## 步骤3：录入

用户确认后，调用record.py批量录入：

```bash
/c/Users/cha18/AppData/Local/Programs/Python/Python312/python.exe -X utf8 "F:/学习/串串香支出与利润/record.py" batch <日期> 支出 "子类:金额:备注" ...
```

## 注意事项

- 图片通常是微信临时文件，路径可能过期，应及时识别
- 子类映射仅供参考，用户可能需要修正
- 如果OCR识别不准，以用户口头描述为准
- 金额规则：小数四舍五入到元
