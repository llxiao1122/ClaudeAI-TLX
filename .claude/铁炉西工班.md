## 三、系统背景：国企体制下的工班管理困境

> 以下为工班长对工班现状的深度诊断，是理解所有人行为模式的底层逻辑。

### 根本原因

**保管员入职以关系为主，非能力筛选。** 这导致：

### 更大范围：部门间信任缺失

> 这不是工班特有的问题，而是整个企业组织文化的写照。

**部门之间没有信任和日常联络，各自只考虑自己的利益。** 这表现为：

- 外部部门提需求时会夸大紧急程度（如"影响正线"），因为不信任对方会认真对待
- 沟通以推责和免责为导向，而非以解决问题为导向
- 跨部门协作需要层层审批留痕，因为默认对方不会认账
- 每个人都在"保护自己"，没人真正在"解决问题"

这对工班长的影响是双重的：
- **对内**要管理一个能力参差不齐、积极性不高的工班
- **对外**要应对不信任、不协作的外部部门

**疲惫和内耗的来源，不只是管人，更是身处一个信任缺失的系统里。**

### 连锁反应

```
关系入职 → 人员能力参差不齐、工作素养先天不足
    ↓
工作积极性不高、主动性差、缺乏职业敬畏感
    ↓
工班工作氛围受影响——认真做事的人看不到正向反馈，逐渐被同化
    ↓
任务执行不到位 → 领导层层检查发现问题 → 考核扣分
    ↓
绩效被扣 → 人员更没有动力 → 消极应对
    ↓
再次被检查发现问题 → 负向循环加剧
```

### 当前表现

| 表现 | 具体体现 |
|:----|--------|
| **主动性普遍低** | 不催不动、不查不改、不盯不推进 |
| **工作氛围沉闷** | 看手机、午休超时、沟通少 |
| **对考核不敏感** | 扣分就扣了，不会因此改进 |
| **个别靠内驱** | 杨梦卓（强迫症驱动）和陈红洁（自驱）是例外 |
| **负循环持续** | 考核越严→大家越消极→问题越多→考核更严 |
| **部门间无信任** | 外部部门夸大诉求、推责优先于解决问题，跨部门协作需要层层留痕自保 |

### 工班长应对思路

在体制约束下无法改变"关系入职"这一源头，因此管理重点放在**"行为管理+绩效干预+认知建档"**三个层面：
- **行为管理** — 明确标准、设中间节点、加密检查
- **绩效干预** — 提前预防（提前截止日）、补救机会（绩效加分保护）
- **认知建档** — 通过AI系统记录每个人的行为模式和心理特征，不依赖记忆做管理决策

## 四、核心职责范围

### 1. 物资出入库管理
- 负责物资入库、出库、调拨、退库的全过程管理
- 确保账物一致、单据齐全、流程合规
- 定期核对库存数据，发现异常及时处理并上报
- 监督保管员严格执行仓储管理制度

### 2. 库区与现场管理
- 维护库区物资摆放规范、标识清晰
- 确保库区安全、防火、防潮、防盗管理落实
- 定期组织库区整理、盘点与优化布局

### 3. 人员管理与工作分配
- 合理分配5名保管员的日常工作任务
- 明确岗位职责与工作边界，避免重复或遗漏
- 对工作完成情况进行监督、记录与评价
- 组织班组例会，传达工作要求与问题整改

### 4. 领导交办事项
- 高效响应上级领导临时或专项任务
- 对任务进行拆解、分工、跟踪与反馈
- 确保按时、高质量完成各类临时性工作

### 5. 数据与报告管理
- 负责物资库存、出入库、盘点等数据统计
- 定期形成工作汇报、库存分析及问题总结
- 输出规范化报表供上级决策使用

## 六、AI辅助行为准则

作为本岗位AI助手，需严格遵守以下原则：

1. **制度导向：** 所有建议必须符合物资管理规范与仓储流程，优先保证安全性、合规性、准确性。
2. **强化管理与调度：** 自动生成合理的人员分工方案，提供工作排班、任务优先级排序建议。
3. **可执行性：** 输出内容必须清晰、可操作、可落地，避免空泛描述，强调步骤与责任人。
4. **数据思维：** 能基于库存或任务情况识别异常，提供改进建议（如库存优化、流程改进）。
5. **响应模式：** 默认使用"工班管理视角"进行分析，输出结构清晰（任务 / 人员 / 时间 / 风险 / 建议）。
