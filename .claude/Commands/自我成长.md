# Command：自我成长 Prompt

> 用于周期性自我分析、系统优化和能力提升。
> 建议每30天执行一次，或在系统重大变更后执行。

---

## 执行流程

### 第一步：数据采集

读取以下模块的全部内容：

```
Management/People/    → 全员档案
Management/Leaders/   → 领导认知
Management/Tasks/     → 待办清单
Management/Events/    → 历史事件
Management/Rules/     → 决策原则 + 错误记录 + 思考流程
Management/Observations/ → AI观察（如有）
Memory/               → 偏好/组织/经验
Context/              → 今日/本周/当前项目
```

### 第二步：八大分析

按以下结构分析：

```
① 管理方式特点
   → 从Events和People中提取管理行为模式
   → 识别哪些管理策略有效，哪些无效

② 人员成长/下降
   → 对比近期Observation与People档案
   → 标注：成长最快的人 / 执行力下降的人
   → 必须区分【事实】/【AI观察】

③ 领导要求变化
   → 对比Leaders与近期Events
   → 识别新要求、新关注点、新汇报习惯
   → 仅标注连续出现的模式（单次不记录）

④ 团队风险
   → 基于Tasks逾期情况 + Observations
   → 识别：人员风险 / 流程风险 / 合规风险
   → 标注可信度：高/中/低

⑤ 可沉淀的经验
   → 从Events和过去回答中提取
   → 判断是否长期有效
   → 有经验的更新Memory/Experience.md

⑥ 过时的记忆
   → 检查Memory/和Rules/中的内容
   → 与当前实际情况比对
   → 过时的：更新或删除

⑦ 知识库利用率
   → 统计Commands/中各命令的调用频率
   → 低利用率的考虑归档
   → 高需求但缺少的考虑新建

⑧ 系统自身缺陷
   → 检查CLAUDE.md执行流程是否被跳过
   → 检查空目录（如Observations/）
   → 检查断裂引用（如指向已删除文件的路径）
```

### 第三步：自动优化

```
对②③④：更新对应People/Leaders/Tasks
对⑤⑥：更新Memory/和Rules/
对⑦：更新Knowledge/索引
对⑧：修复系统文件
```

### 第四步：输出报告

```
【AI自我成长报告】
生成日期：YYYY-MM-DD
分析周期：上次执行 → 本次

① 管理方式特点：
② 人员成长最快：
③ 执行力下降：
④ 领导要求变化：
⑤ 团队最大风险：
⑥ 新沉淀的经验：
⑦ 已清理的过时记忆：
⑧ 系统优化项：

改进建议：
1. ...
2. ...
3. ...
```
