# Workflow：制度查询与分析

> 查询知识库制度并给出结构化分析的标准流程

---

## 输入

- 用户问题（关于某项制度/流程/规范）
- 适用场景（验收/考核/安全/出入库等）

## 流程

### Step 1：理解需求
```
确定用户在问什么：
  → 是查具体规定？还是流程步骤？还是责任归属？
  → 涉及哪个领域（物资/安全/危废/消防等）？
```

### Step 2：索引定位
```
读 知识库/INDEX.md → 按关键词定位制度文件
  → 例如搜"验收" → "01/02/2024-物资验收入库管理规定-A1"
  → 例如搜"消防" → "02/科室发文" + "03/消防管理指导手册"
```

### Step 3：读取文件内容
```
读定位到的文件
  → 提取相关章节
  → 注意版本号（以01-主目录中的最新版为准）
```

### Step 4：交叉验证
```
检查关联文件：
  → INDEX.md中标注的"关联"字段
  → 如查"仓储"→检查"验收规定+考核细则+巡查手册"
  → 版本冲突时按 decision.md 规则处理
```

### Step 5：判断时效性
```
查文件日期/版本号：
  → 2024年后的文件 → 有效
  → 科室发文2022-2023 → 检查是否有2024-2025修订版
  → 旧版已归档到10-的 → 标注旧版并引用新版
```

### Step 6：生成回答

结构：
```
制度名称：[名称-版本号]
适用场景：[什么情况下用]
核心规定：[关键条款]
注意事项：[例外/特殊要求]
关联制度：[相关制度清单]
```

### Step 7：标注来源
```
引用格式：[制度名称-版本号，章节号]
例如：[物资仓储管理办法-B1版，§5.9]
科室发文：[调配综文〔2023〕XX号]
```

---

## 检查清单

- [ ] 是否定位到了正确的制度文件？
- [ ] 是否检查了版本号（最新版优先）？
- [ ] 是否检查了关联制度？
- [ ] 2024年前的制度是否确认没有被替代？
- [ ] 引用是否标注了来源？
- [ ] 不确定的内容是否标注了【推测】/【需确认】？
