# 管理经验

## 人员管理
- 苗笑天：半日节点+正向激励有效，怕被说→有隐瞒倾向，问题需私下反馈
- 苗笑天（补充7/7后）：**能接受直面批评，甚至反应更积极**——私下坚定谈话后主动组织分工、答疑态度好，展现出被激活的领导力。策略应调整为：直接沟通+坚定期望+正反馈跟进，而非完全避免批评
- 苗笑天（补充7/8）：**上升势头与旧习惯并存**——7/2安排的照片反馈拖6天未做，说明执行力改善尚未完全固化。旧模式（拖延→隐瞒）仍然可能复发，需持续半日节点控制直到新习惯巩固
- 杨梦卓：能力强但骄傲，不适合公开批评，私下先肯定再提建议
- 谭继衡：责任感驱动，谈话沟通有效，需明确截止+信任放权
- 张志斌：需明确指令清单+半日检查，被动但执行不差

## 沟通
- 领导安排：拆解为表层任务/隐含目标/风险重点/验收标准四层
- 汇报：按领导类型选择策略（稳定推进型→按节点汇报）

## 系统建设
- **三层数据模型**（台账提醒系统验证有效）：Knowledge/（原始数据层·人阅读）→ Commands/脚本（自动化层·机器执行）→ Memory/+context/（执行上下文层·AI每日调用）。核心理念：修改底层数据时只需更新Knowledge和脚本，上层自动联动。适用于任何"规则数据→定期提醒"类场景
- **跟进闭环触发机制**：非紧急但重要的事项（如苗笑天照片拖6天）需要定时触发点，不能依赖人脑记忆。可扩展为"每周五扫一遍tasks.md中标记⏳超过3天的非紧急事项"

## 标准建设
- **执行标准盲区教训**（迎检暴露）：日常标准容易只覆盖"看得见"的地方（主通道）而遗漏隐蔽区域（货架下/托盘间/外墙腰线/靠墙处）。根源不是执行力，是检查标准设计有结构性盲区。教训：任何周期性检查标准，必须单独列"隐蔽区域项"，并明确责任人+频率
