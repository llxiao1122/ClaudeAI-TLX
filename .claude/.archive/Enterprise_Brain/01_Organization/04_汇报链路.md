# 汇报链路

> 正式汇报关系、事项汇报规则、沟通策略。
> 来源：Memory/Organization.md, Management/Leaders/, Management/Experience.md

---

## 一、正式汇报链

```
郝晓平(总经理)
  └── 商丽娜(分管物资+财务)
        └── 李彦奎(物资部长)
              └── 张新宇(副部长)
                    └── 王敬宇(主任·业务)
                          ├── 卢丽英(副主任·安全) ← 安全相关
                          └── 李林骁(工班长)
                                └── 5名保管员
```

## 二、按事项汇报路径

| 事项 | 汇报路径 | 说明 |
|:----|:---------|:-----|
| **日常业务** | 保管员→李林骁→王敬宇 | 逐级上报 |
| **安全事项** | 李林骁(兼职安全员)↔王亮+卢丽英<br>陈红洁(待批)→协助执行 | 工班长为安全第一责任人，陈红洁协助巡查 |
| **包保事项** | 董文静→李林骁→保管员(传达) | 包保人转达上级要求 |
| **资产/废旧** | 苗笑天→李林骁→荆幸斌 | 材料棚-工班-资产组 |
| **内勤/公文** | 张梦圆→陈红洁→李林骁(如需) | 内勤对接线 |
| **建设移交** | 谭继衡→李林骁→靳攀(每日16:00) | 每日进度更新 |
| **安全隐患/事件** | 发现者→李林骁(紧急同时报) → 王敬宇+卢丽英 | 安全事件24h内上报 |

## 三、需向上汇报的事项

| 触发条件 | 汇报对象 | 时限 |
|:---------|:---------|:----:|
| 上级检查结果 | 王敬宇 | 检查后即时 |
| 安全事件/隐患 | 王敬宇+卢丽英 | 即时（紧急）/ 24h（一般） |
| 物资盘亏/差异 | 王敬宇 | 发现后当日 |
| 制度执行异常 | 王敬宇 | 发现后 |
| 重大设备故障 | 王敬宇→张新宇 | 即时 |
| 迎检准备情况 | 王敬宇 | 检查前 |

## 四、各领导沟通策略

| 领导 | 标签 | 汇报策略 |
|:----|:-----|:---------|
| 王敬宇 | 稳定推进型 | **按节点汇报**，偏差主动预警，不催不意味着不重要 |
| 卢丽英 | 安全响应型 | **安全相关优先响应**，有整改立即反馈 |
| 商丽娜 | 宏观指导型 | 保持信息通畅，不强制极短时限，跨部门检查时重点准备 |
| 李彦奎 | 安全高标准型 | 安全事项高标准应对，检查时细节不放松 |
| 张新宇 | 操作规程型 | 操作规程类事项重点准备，运输设备台账等 |
| 董文静 | 时限催促型 | 转达"今天下班前"类要求时立即响应 |
| 靳攀 | 规范推进型 | 每日16:00进度更新，定时定点 |
| 荆幸斌 | 稳定推进型 | 群通知@+截止+回复，按时完成即可 |

## 五、汇报四层拆解法

领导安排任何任务时，应拆解为：

```
表层任务：领导说了什么
隐含目标：领导真正想要什么
风险重点：最容易出问题的地方
验收标准：怎样才算做完
```
