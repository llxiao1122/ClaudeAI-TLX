## 增量学习流程

### 输入触发条件

| 触发源 | 说明 |
|:-------|:-----|
| git 新增文件 | `Knowledge/` 出现未跟踪的 .doc/.docx/.pdf |
| git 修改文件 | 已有制度文件被替换/更新 |
| .claude 事件记录 | 新 Observations/Events 涉及制度变更 |
| 用户主动告知 | "新增了一批制度文件" |

### 处理步骤

```
1. 差异扫描
   └── 对比当前 Knowledge/ 与 04_Evolution/基线记录
       └── 列出: 新增文件 / 修改文件 / 版本变化

2. 文件分类
   └── 制度类(01/02/03) → 进入版本分析
   └── 手册类(03) → 进入操作层分析
   └── 发文类(02-02) → 判断替代/新增
   └── 非制度类(04-09) → 仅索引更新

3. 版本链判断
   └── 是否替代已有制度? → 更新 03_Management/制度版本管理.md
   └── 是否全新制度? → 新增条目
   └── 是否科室发文? → 判断替代 vs 补充

4. 管理逻辑提炼
   └── 制度目的 → 是否需要更新 management_purposes.md
   └── 核心规则 → 是否需要更新 供应链/安全/台账 规则链
   └── 审批链 → 是否需要更新 权责关系/审批逻辑
   └── 风险控制 → 是否需要更新 风险识别

5. 认知层合并
   └── 新规则 → 写入对应认知文件
   └── 旧规则废弃 → 标记+移入历史
   └── 冲突 → 标记到 制度版本管理.md 冲突表

6. 索引更新
   └── Enterprise_Brain.md 关键词索引
   └── knowledge_links.md 速查表

7. 变更记录
   └── 写入 04_Evolution/企业大脑演化记录.md
```

### 判断规则

| 信号 | 判定 | 动作 |
|:-----|:-----|:------|
| 文件名含 `修订版` `A1→A2` `B1→B2` | 版本替代 | 标记旧版已替代，更新版本表 |
| 文件名含 `试行` | 过渡性文件 | 标记"试行"，持续关注正式版 |
| 科室发文编号更大 | 补充或替代 | 查内容判断替代 vs 并行 |
| 文件名无年份无版本号 | 来源不明 | 标记 **需人工确认** |
| 与现有制度同名但路径不同 | 潜在冲突 | 记入冲突表 |
