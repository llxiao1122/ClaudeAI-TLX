| 场景 | 汇报方式 | 时限 |
|:-----|:---------|:----:|

| 安全事件 | 即时报告卢丽英/王亮 | 24h内 |

### 汇报层级规律

```
整改类：收到→执行中(如有偏差预警)→完成闭环汇报
日常类：按节点推进→定时同步
紧急类：即时上报→持续跟进→闭环
```

## 五、常见工作要求模式

### 模式1：检查→整改→闭环

```
领导实地检查发现 → 下发整改任务清单 → 工班执行 → 定时汇报 → 领导确认
```
示例：郝总郑东检查 → 商总5项整改 → 一天内两次钉钉会

### 模式2：约谈/事件→专项部署

```
外部事件触发 → 专项会议部署 → 全面排查 → 重点整治 → 升级管控
```
示例：郑州约谈 → 迎检专项会议 → 安全+卫生+台账三线整治

### 模式3：定期巡查→问题反馈

```
定期巡查 → 发现问题 → 提出整改要求(附件据) → 设定时限 → 跟踪闭环
```
示例：王敬宇4月中州大道/6月铁炉西

### 模式4：日常对接→节点推动

```
日常对接人传达 → 明确截止时间 → 完成反馈
```
示例：董文静传达"今天下班前"、靳攀"每日16:00"

### 领导对执行的标准要求

| 标准 | 具体表现 |
|:-----|:---------|
| **可视化** | 规格明确到尺寸/字体/位置（A4/130号宋体加粗居中） |
| **全面性** | 同类问题全面排查，不修单点 |
| **可追溯** | 巡查记录到人、到库区、到时段 |
| **及时性** | 当天闭环、"今天务必"、下班前汇报 |
| **态度端** | 隐蔽区域体现态度，不突击应付 |

## 六、处理问题偏好

### 问题上报后的预期处理路径

| 问题类型 | 预期处理 | 领导角色 |
|:---------|:---------|:---------|
| 现场缺陷 | 立行立改，拍照闭环 | 检查者→验收者 |
| 安全隐患 | 立即整改+同类排查+制度补漏 | 部署者→追责者 |
| 台账差异 | 逐项核对，查清原因 | 监督者→验收者 |
| 人员问题 | 追责不到位者（工班长/直接责任人） | 追责者 |

### 可验证的管理原则

| 原则 | 证据 |
|:-----|:-----|
| **程序合规优先** | 检查引用制度文件编号 |
| **节点管控** | 一天两次汇报、每日16:00定时 |
| **责任到人** | 巡查记录必须注明每人负责库区 |
| **不突击不造假** | "禁止检查组到时突击打扫" |
| **全面排查非单点** | 明线/指示灯/CIMS隐患均要求"全面排查" |
| **标准量化** | 物料盒A4/130号宋体加粗、每日16:00 |
| **闭环管理** | 问题→整改→汇报三阶段缺一不可 |
| **问责逐级** | 现场问题→直接责任人→追责工班长→上升 |
