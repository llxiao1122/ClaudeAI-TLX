## 使用指引

### 检索策略

| 查询类型 | 路径 |
|:---------|:-----|
| 找制度(规定/办法/细则) | `Operations/04_模块管理目的.md` 定位模块 → Knowledge/ 对应目录 |
| 找操作(怎么做/流程) | `Operations/05_知识关联索引.md` 找手册 → Knowledge/03-工作指导手册/ |
| 找安全相关 | `Operations/02_安全责任链.md` 定方向 → 制度+手册双重验证 |
| 找台账(表格/清单) | `Operations/03_台账管理矩阵.md` 定节奏 → Knowledge/04-工作台账与工具/ |
| 确认版本新旧 | `Operations/06_文件版本状态.md` + `03_Management/03_制度版本管理.md` |

### 认知原则

| 原则 | 含义 |
|:-----|:------|
| 三重证据 | 制度=约束，手册=操作，台账=证据 |
| 安全平行线 | 安全管理平行于业务管理的第二体系，在危废/消防/特种设备处交叉 |
| 工班长枢纽 | 工班长是连接"管理要求"与"现场执行"的唯一枢纽 |
| 最终目的 | 物资可追溯、安全无事故、账实相一致 |

### 安全边界

- ❌ 不要直接复制 Knowledge/ 原始文件内容作为回答
- ✅ 用 Enterprise_Brain/ 指引查询方向
- ❌ 不要对未读过的二进制文件内容做推断
- ❌ 不要修改 Knowledge/ 的文件结构
- ✅ Enterprise_Brain/ 可随认知深化更新
