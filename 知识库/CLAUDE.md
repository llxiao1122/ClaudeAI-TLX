# 铁炉西工班知识库 — AI使用与维护规则

> 本文档定义AI如何阅读、检索、引用、维护本知识库。
> 所有AI助手在回答铁炉西工班相关问题时，应优先遵守以下规则。

---

## 一、AI阅读知识库的方式

### 1.1 首次接入
1. **先读 INDEX.md** — 了解全量文件索引、分类、关键词
2. **再读具体文件** — 根据用户问题定位到具体文件
3. **关联查找** — INDEX.md中标注了"关联"字段，可找到相关制度

### 1.2 回答优先级
```
用户提问
  ┣━━ 先查 知识库/ INDEX.md → 定位文件
  ┣━━ 再读对应文件内容 → 获取准确信息
  ┣━━ 关联制度交叉验证 → 确保回答完整
  ┗━━ 标注引用来源 → 格式：[制度名称-版本号]
```

### 1.3 引用格式
- 引用制度时标注：`[制度名称-版本号，章节号]`
- 例如：`[物资仓储管理办法-B1版，§5.9]`
- 引用科室发文时标注：`[调配综文〔2023〕XX号]`
- 引用指导手册时标注：`[指导手册名称，章节号]`

### 1.4 置信度标注
| 情况 | 标注 |
|------|------|
| 制度原文明确规定 | 直接引用，不标注 |
| AI推断/关联建议 | 标注**【AI建议】** |
| 多个版本有冲突 | 标注**【版本冲突，需人工确认】** |
| 无法找到依据 | 标注**【知识库无直接依据】** |

## 三、版本管理规则

### 3.1 制度版本优先顺序
```
最新版（01-物资管理制度/） > 科室发文（02-安全管理/02-） > 旧版（10-旧版与归档/）
```

### 3.2 已知版本替代关系

| 制度 | 最新版 | 旧版状态 |
|------|--------|---------|
| 物资仓储管理办法 | B1版（02-仓储与验收） | A5版 → 已归档 |
| 物资验收入库管理规定 | A1版（02-仓储与验收） | 2024版 → 已归档 |
| 捐赠物资管理细则 | A1版（03-考核与细则） | 2024版 → 已归档 |
| 修复件和周转件管理细则 | A0版（03-考核与细则） | 2024版 → 已归档 |

### 3.3 科室发文版本关系
- `2022-2023`年科室发文→ 基础要求
- `2024-2025`年科室发文→ 修订/补充（更新版）
- 同一主题以**最新文号**为准（如危废：2022-19号 → 2023-40号修订）

## 四、新增资料处理规范

### 4.1 新增文件分类规则

新增文件时，根据内容判定所属分类：

| 文件内容 | 放入目录 |
|---------|---------|
| 物资管理制度（仓储/验收/考核/采购等） | `01-物资管理制度/` 对应子目录 |
| 安全制度/安全通知/安全手册 | `02-安全管理/` 对应子目录 |
| 操作手册/作业指导书/流程说明 | `03-工作指导手册/` 对应子目录 |
| 台账模板/检查表/值班表/工作工具 | `04-工作台账与工具/` |
| 培训PPT/学习资料 | `05-培训与学习资料/` 对应子目录 |
| 公文模板/公文规范 | `06-公文模板与规范/` |
| 6S/卫生/标识/货位标准 | `07-现场管理与6S/` |
| 专项工作（如三菱） | 新建 `XX-专项名称/` |
| 评分表/评估表 | `09-评估表/` |
| 国家标准/行业标准/法律法规 | `11-法规标准/` |

### 4.2 新文件命名规则
```
格式：[年份]-[业务描述]-[版本号].[ext]
示例：2026-防汛工作管理补充通知-24号.pdf
示例：2026-物资仓储管理办法-C1版.doc
```

**禁止出现的命名词汇：** `最终版` `最新版` `最终最终版` `修改版` `副本` `复制` `New` `New(1)` `(1)`

### 4.3 新文件必须更新

新增文件后，必须同步更新：
1. `INDEX.md` — 在对应分类下新增条目
2. 对应目录的 `README.md` — 如有文件清单则新增

## 五、发现重复文件的处理

### 5.1 去重判断标准
```
MD5一致 → 内容完全相同 → 保留一个，另一个记录
版本号不同 → 保留最新版本，旧版归档
同名不同版 → 比较版本号+日期，保留新版
```

### 5.2 处理流程
1. **标记**：在`INDEX.md`对应条目标注`⚠️ 疑似重复`
2. **记录**：写入`99-知识库系统/整理日志/`
3. **不删除**：原始文件永远不删除，只有标记和归档
4. **通知**：在对话中告知用户建议处理方式

## 六、知识库维护周期

| 周期 | 维护动作 |
|:----:|---------|
| 每季度 | 检查科室发文是否有新版，更新INDEX |
| 制度更新时 | 旧版→归档，新版→主目录，更新INDEX+版本对照 |
| 新增专项时 | 新建专项目录，更新README+INDEX+CLAUDE.md |
| 每年初 | 清理过期台账（如值班表/灭火器计划），更新年份 |

---

## 七、安全与边界

1. **不要删除任何原始文件** — 所有操作基于"复制+重组"，原始目录永不修改
2. **不要修改Office/PDF文件内容** — 知识库文件为只读引用
3. **AI不可替代人工确认** — 涉及考核、奖惩、责任判定的回答，需标注引用制度原文
4. **注明参考来源** — 所有引用必须标注具体制度名称和版本号
5. **超出知识库范围** — 清晰告知"该问题不在当前知识库范围内"

## 八、与主项目CLAUDE.md的关系

本知识库的CLAUDE.md 与 项目根目录 `CLAUDE.md` 分工不同：

| 文件 | 范围 |
|------|------|
| `CLAUDE.md`（根目录） | 项目管理：双设备同步、人员速查、日常入口 |
| `知识库/CLAUDE.md`（本文） | 知识管理：AI如何使用知识库、分类规则、版本管理 |

> AI在回答时应同时参考两个文件，项目管理问题查根目录CLAUDE.md，知识查询问题查知识库INDEX.md和本CLAUDE.md。
