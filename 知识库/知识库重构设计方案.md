## 二、新知识库架构

### 2.1 设计原则

1. **功能导向分类** — 按业务领域（而非文件来源）分类
2. **扁平化结构** — 最多2层嵌套，避免深层目录
3. **自解释命名** — 文件名 = 日期-制度名称-版本号
4. **虚拟索引层** — INDEX.md作为统一入口，AI仅需读取这一个文件即可定位
5. **原文件不动** — 所有原始文件原位置保留，新结构通过复制/链接建立
6. **可扩展** — 新增文件按分类规则直接放入对应目录

### 2.3 统一命名规范

**规范格式：** `[年份]-[业务分类]-[文件名称]-[版本号].[ext]`

| 元素 | 规则 | 示例 |
|:----|------|------|
| 年份 | 4位数字，未知年份省略 | `2024-` |
| 业务分类 | 2-4字中文缩写，可选 | `仓储-验收-` |
| 文件名称 | 清晰自解释，去（1）/副本等 | `物资验收入库管理规定` |
| 版本号 | A0/B1/C2格式，可选 | `-A1` |
| 分隔符 | 全用`-`连字符，无空格 | ✅ |
| 禁止词汇 | 最终版、最新版、最终最终版、修改版、副本、复制、New、New(1) | ❌ |

**特殊约定：**
- 科室发文：`[年份]-[主题]-[文号后缀].pdf` (如 `2023-消防器材巡查标准通知-13号.pdf`)
- 模板文件：`[名称]-模板.[ext]` (如 `值班表模板-2026.xlsx`)
- 待确认文件：`【待确认】-[原文件名].[ext]`
